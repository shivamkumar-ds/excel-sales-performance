# 2. Date Functions

## 2.1 EOMONTH

**Purpose**  
Returns last day of month after adding months.

**Syntax**
```

=EOMONTH(start_date, months)

```

**Example**
```

=EOMONTH(C5,2)

```

- `months = 0` → end of current month  
- `months = -1` → end of previous month  
- `MOD(3 - MONTH(date), 3)` to shift the given date to the last month of its calendar quarter (adding 0, 1, or 2 months as needed).


**Used for**
- Month-end reporting  
- Quarter-end calculations  
- Financial modeling  

---

## 2.2 WORKDAY.INTL

**Purpose**  
Adds working days with custom weekend definition.

**Syntax**
```

=WORKDAY.INTL(start_date, days, weekend, [holidays])

```

**Arguments**
- `start_date`
- `days`
- `weekend` → code OR 7-digit string
- `holidays` → optional range

**Weekend String Rule**
- 7 digits starting from Monday  
- `0` = working day  
- `1` = weekend  

Example:
```

"0001011"

```
Thursday, Saturday, Sunday are weekends.

---

## 2.3 EDATE

**Purpose**  
Adds or subtracts months.

**Syntax**
```

=EDATE(start_date, months)

```

Example:
```

=EDATE(A1,2)

```

**Used in**
- Loan schedules  
- Payment tracking  
- Financial forecasting  

---

## 2.4 DATEDIF

**Purpose**  
Returns difference between two dates.

**Syntax**
```

=DATEDIF(start_date, end_date, unit)

```

**Units**
- `"Y"` → years  
- `"M"` → months  
- `"D"` → days  
- `"MD"` → leftover days after removing years & months

# 4. Financial Functions

## 4.1 FV (Future Value)

**Purpose**  
Calculates value of investment in future.

**Syntax**
```

=FV(rate, nper, pmt, [pv], [type])

```

- `rate` → per period rate  
- `nper` → total periods  
- `pmt` → regular payment (negative)  
- `pv` → initial investment  
- `type` → 0 end (default), 1 beginning  

---

## 4.2 PV (Present Value)

**Purpose**  
Amount required today for a future goal.

**Syntax**
```

=PV(rate, nper, pmt, [fv], [type])

```

---

## 4.3 PMT (Payment)

**Purpose**  
Calculates loan or investment payment.

**Syntax**
```

=PMT(rate, nper, pv, [fv], [type])

```

- Payments show negative (cash outflow)
- Add minus before formula if you want positive display

# 5. Loan Schedule Logic

**Rate per period**
```

Annual rate / periods per year

```

**Total periods**
```

Loan term * periods per year

```

**Interest**
```

Previous balance * rate

```

**Principal**
```

Payment - Interest + Extra payment

```

**New balance**
```

Previous balance - Principal

```

Extra payment reduces total interest.

Interest savings:
```

Projected interest - Actual interest

```

# 6. Capital Budgeting

## 6.1 NPV

**Meaning**  
Present value of inflows − outflows

**Syntax**
```

=NPV(rate, cashflows) - Initial Investment

```

Important:
- NPV assumes first cash flow at end of period
- Initial investment at time 0 must be subtracted manually

Decision rule:
- Accept if NPV > 0

---

## 6.2 IRR

**Meaning**  
Rate where NPV = 0

**Syntax**
```

=IRR(values, [guess])

```

- Include ALL cash flows including initial investment
- Accept if IRR > Cost of Capital
- Conventional cash flow → single sign change → unique IRR

# 7. Depreciation

## Key Terms
- Cost
- Salvage value
- Life
- Depreciation amount
- Book value = Beginning value − Depreciation

---

## 7.1 Straight Line (SLN)

Equal depreciation every year.

```

=SLN(cost, salvage, life)

```

Math:
```

(cost - salvage) / life

```

---

## 7.2 SYD (Sum of Years Digits)

Higher depreciation in early years.

```

=SYD(cost, salvage, life, period)

```

Example (10 years):
- Sum = 55
- Year 1 → 10/55
- Year 2 → 9/55

---

## 7.3 DDB (Double Declining Balance)

Accelerated method.

```

=DDB(cost, salvage, life, period, [factor])

```

- Default factor = 2
- May not end exactly at salvage value

---

# 8. Interest Concept

Interest → amount paid  
Interest rate → percentage charged  

Effective periodic rate:

$$
(1 + r)^{1/n} - 1
$$

Where:
- r = annual rate  
- n = compounding periods
